# 开发启动说明

## Gateway

进入后端目录：

```powershell
cd gateway
```

首次创建虚拟环境：

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

真实模型能力需要在 `gateway/.env` 中配置。可以参考 `gateway/.env.example`，不要把真实 Key 写进代码。

中国境内部署建议 STT 主链路用独立 SenseVoiceSmall HTTP 服务：

```text
STT_PROVIDER=sensevoice_http
SENSEVOICE_URL=http://127.0.0.1:9001/transcribe
SENSEVOICE_LANGUAGE=zh
SENSEVOICE_TIMEOUT_SECONDS=60
```

中国境内部署建议 LLM 主链路用 Qwen，DeepSeek 做备用/推理增强，Ollama 做本地离线兜底，OpenAI 作为海外或高级能力备用：

```text
LLM_PROVIDER=qwen
LLM_SYSTEM_PROMPT=你是一个温和、简洁、可靠的桌面 AI 机器人助理。回答要自然，适合通过语音朗读。

QWEN_API_KEY=sk-your-qwen-key
QWEN_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
QWEN_LLM_MODEL=qwen-plus

DEEPSEEK_API_KEY=sk-your-deepseek-key
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_LLM_MODEL=deepseek-chat

OLLAMA_BASE_URL=http://127.0.0.1:11434
OLLAMA_LLM_MODEL=qwen2.5:7b
OLLAMA_TIMEOUT_SECONDS=60
```

OpenAI 仍保留为可选备用：

```text
OPENAI_API_KEY=sk-your-api-key
STT_PROVIDER=openai
OPENAI_STT_MODEL=gpt-4o-mini-transcribe
OPENAI_STT_LANGUAGE=zh
LLM_PROVIDER=openai
OPENAI_LLM_MODEL=gpt-5.4-mini
OPENAI_LLM_INSTRUCTIONS=你是一个温和、简洁、可靠的桌面 AI 机器人助理。回答要自然，适合通过语音朗读。
TTS_PROVIDER=openai
OPENAI_TTS_MODEL=gpt-4o-mini-tts
OPENAI_TTS_VOICE=alloy
OPENAI_TTS_SAMPLE_RATE=24000
```

如果不创建 `.env`，Gateway 会继续使用 `mock` Provider，便于离线调试。

启动服务：

```powershell
.\.venv\Scripts\python.exe -m uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

也可以在项目根目录使用脚本启动：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts/start_gateway.ps1 -Reload
```

停止服务：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts/stop_gateway.ps1
```

健康检查：

```powershell
Invoke-RestMethod http://127.0.0.1:8000/health
```

## SenseVoice STT Service

SenseVoiceSmall 作为独立 HTTP 服务运行，默认监听：

```text
http://127.0.0.1:9001/transcribe
```

首次安装可选依赖：

```powershell
cd gateway
.\.venv\Scripts\python.exe -m pip install -r requirements-sensevoice.txt
```

启动 SenseVoice 服务：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts/start_sensevoice.ps1
```

健康检查：

```powershell
Invoke-RestMethod http://127.0.0.1:9001/health
```

切换 Gateway 到 SenseVoice：

```text
STT_PROVIDER=sensevoice_http
SENSEVOICE_URL=http://127.0.0.1:9001/transcribe
SENSEVOICE_LANGUAGE=zh
SENSEVOICE_TIMEOUT_SECONDS=60
```

SenseVoice 服务可选环境变量：

```text
SENSEVOICE_MODEL=iic/SenseVoiceSmall
SENSEVOICE_DEVICE=cpu
SENSEVOICE_HUB=ms
SENSEVOICE_USE_ITN=true
SENSEVOICE_BATCH_SIZE_S=60
```

如果有 NVIDIA GPU，可以尝试：

```text
SENSEVOICE_DEVICE=cuda:0
```

本地文件测试：

```powershell
curl.exe -X POST http://127.0.0.1:9001/transcribe `
  -F "language=zh" `
  -F "audio=@gateway/app/static/audio/input/your_audio.wav;type=audio/wav"
```

## CosyVoice TTS Service

CosyVoice2 作为独立 HTTP 服务运行，默认监听：

```text
http://127.0.0.1:9002/synthesize
```

当前项目先提供 placeholder HTTP 服务，用于验证 Gateway 到 TTS 服务再到 CoreS3 播放的协议链路。真实 CosyVoice2 推理后续在这个服务里替换实现。

启动 CosyVoice 服务：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts/start_cosyvoice.ps1
```

健康检查：

```powershell
Invoke-RestMethod http://127.0.0.1:9002/health
```

切换 Gateway 到 CosyVoice：

```text
TTS_PROVIDER=cosyvoice_http
COSYVOICE_URL=http://127.0.0.1:9002/synthesize
COSYVOICE_VOICE=default
COSYVOICE_SAMPLE_RATE=22050
COSYVOICE_TIMEOUT_SECONDS=120
```

CosyVoice 服务可选环境变量：

```text
COSYVOICE_SERVER_MODE=placeholder
COSYVOICE_MODEL=iic/CosyVoice2-0.5B
COSYVOICE_REPO=C:\path\to\CosyVoice
COSYVOICE_OUTPUT_DIR=app/static/audio/cosyvoice
```

支持的服务模式：

```text
placeholder
cosyvoice2_zero_shot
cosyvoice_sft
```

`placeholder` 用于协议烟测，不需要真实 CosyVoice 依赖。

`cosyvoice2_zero_shot` 用于 CosyVoice2-0.5B，需要参考音频和参考文本：

```text
COSYVOICE_SERVER_MODE=cosyvoice2_zero_shot
COSYVOICE_REPO=C:\path\to\CosyVoice
COSYVOICE_MODEL=iic/CosyVoice2-0.5B
COSYVOICE_PROMPT_TEXT=这是一段用于克隆音色的参考文本。
COSYVOICE_PROMPT_AUDIO=C:\path\to\prompt.wav
```

`cosyvoice_sft` 用于 SFT 固定音色模型，例如 CosyVoice-300M-SFT：

```text
COSYVOICE_SERVER_MODE=cosyvoice_sft
COSYVOICE_REPO=C:\path\to\CosyVoice
COSYVOICE_MODEL=iic/CosyVoice-300M-SFT
COSYVOICE_VOICE=中文女
```

真实 CosyVoice 推理需要先准备官方 CosyVoice 仓库和依赖。第一轮建议先用 `placeholder` 跑通 Gateway 协议，再切到 `cosyvoice2_zero_shot`。

本地测试：

```powershell

  text = "你好，我是你的桌面机器人。"
  voice = "default"
  sample_rate = 22050
  format = "wav"
```


## Firmware

进入固件目录：

```powershell
cd firmware
```

先编辑 Wi-Fi 和 Gateway 地址：

```text
include/config.h
```

关键配置：

```cpp
#define WIFI_SSID "YOUR_WIFI_SSID"
#define WIFI_PASSWORD "YOUR_WIFI_PASSWORD"
#define GATEWAY_HOST "http://192.168.1.100:8000"
```

CoreS3 使用 ESP32-S3，只支持 2.4GHz Wi-Fi。`WIFI_SSID` 不能填 5GHz-only 的网络名，例如带 `5G` 后缀且只广播 5GHz 的 SSID。

构建：

```powershell
pio run
```

烧录：

```powershell
C:\Users\tim90\.platformio\penv\Scripts\pio.exe run --target upload --upload-port COMx
```

串口监视：

```powershell
C:\Users\tim90\.platformio\penv\Scripts\pio.exe device monitor --port COMx
```

查找串口：

```powershell
C:\Users\tim90\.platformio\penv\Scripts\pio.exe device list
```

也可以用 PowerShell 查看：

```powershell
[System.IO.Ports.SerialPort]::GetPortNames()
```

如果找不到端口，先确认 CoreS3 已经通过 USB 数据线连接电脑。部分线只能充电，不能传数据。

如果找到了端口，例如 `COM7`，烧录命令就是：

```powershell
C:\Users\tim90\.platformio\penv\Scripts\pio.exe run --target upload --upload-port COM7
```

监视串口日志：

```powershell
C:\Users\tim90\.platformio\penv\Scripts\pio.exe device monitor --port COM7 --baud 115200
```

## Audio Chat Smoke Test

当前固件已经支持点击屏幕后固定录音 5 秒，并上传到 Gateway：

```text
IDLE -> LISTENING -> UPLOADING -> SPEAKING -> IDLE
```

烧录最新固件：

```powershell
C:\Users\tim90\.platformio\penv\Scripts\pio.exe run --target upload --upload-port COM3
```

打开串口日志：

```powershell
C:\Users\tim90\.platformio\penv\Scripts\pio.exe device monitor --port COM3 --baud 115200
```

点击 CoreS3 屏幕后，串口里应该能看到类似：

```text
Recording audio...
Recording done: 160044 bytes
POST /audio/chat audio=160044 bytes -> 200
Playing audio: 25644 bytes
```

Gateway 会把音频保存到：

```text
gateway/app/static/audio/input/
```

当前 `/audio/chat` 会完成 mock STT、mock LLM 和 mock TTS，用来验证从录音上传到播放回复音频的完整链路。

当前 STT 已经有服务抽象，支持 `mock`、`sensevoice_http` 和 `openai`。`/audio/chat` 会返回：

```json
{
  "user_text": "这是一次语音上传测试，音频已经成功到达 Gateway。",
  "stt": {
    "provider": "mock",
    "latency_ms": 0,
    "confidence": 1.0
  }
}
```

mock 模式可以在 `gateway/.env` 或启动环境里调整：

```text
STT_PROVIDER=mock
MOCK_STT_TEXT=这是一次语音上传测试，音频已经成功到达 Gateway。
```

真实 STT 使用 `OPENAI_STT_MODEL` 和 `OPENAI_STT_LANGUAGE` 配置。

SenseVoiceSmall HTTP Provider 使用如下约定：

```text
POST http://127.0.0.1:9001/transcribe
Content-Type: multipart/form-data

fields:
  audio: WAV 文件
  language: zh
```

SenseVoice 服务返回至少包含一个文本字段即可，以下任意格式都兼容：

```json
{
  "text": "你好，帮我设置一个提醒"
}
```

```json
{
  "data": {
    "text": "你好，帮我设置一个提醒",
    "confidence": 0.98
  }
}
```

```json
{
  "result": {
    "text": "你好，帮我设置一个提醒",
    "confidence": 0.98
  }
}
```

当前 LLM 已经有服务抽象，支持 `mock`、`qwen`、`deepseek`、`ollama` 和 `openai`。可以直接测试：

```powershell
$body = @{
  user_text = "今天帮我规划一下工作"
  scenario = "deskbot"
} | ConvertTo-Json

Invoke-RestMethod http://127.0.0.1:8000/ai/chat `
  -Method Post `
  -ContentType "application/json" `
  -Body $body
```



DeepSeek：

```text
DEEPSEEK_API_KEY=sk-your-deepseek-key
```

Ollama：


### 上传时串口异常

如果上传时出现类似：

```text
Cannot configure port
PermissionError(13, '连到系统上的设备没有发挥作用。')
```

优先按这个顺序处理：

1. 关闭所有可能占用串口的窗口，包括 `pio device monitor`、Arduino IDE、M5Burner、串口助手、VS Code Serial Monitor。
2. 拔掉 CoreS3 USB，等 3 秒后重新插入。
3. 重新确认端口：

```powershell
C:\Users\tim90\.platformio\penv\Scripts\pio.exe device list
```

4. 使用当前端口重新上传：

```powershell
C:\Users\tim90\.platformio\penv\Scripts\pio.exe run --target upload --upload-port COM3
```

5. 如果卡在 `Connecting...`，按一下 `RESET`；仍不行就按住 `BOOT`，点一下 `RESET`，松开 `BOOT` 后重新上传。

6. 如果仍然报同样错误，换 USB 数据线或 USB 口。这个错误经常由线材、Hub、驱动状态导致。

- `/device/register` 已实际返回设备在线。
- `/device/heartbeat` 已实际写入并返回在线状态。
- Firmware 已通过 PlatformIO 编译。
- `/audio/chat` multipart 上传接口已通过本地烟测。
- Firmware 录音与 `/audio/chat` 上传代码已通过 PlatformIO 编译。
- STT mock Provider 已接入 `/audio/chat`，响应会返回 `user_text` 和 `stt` 信息。
- LLM Provider 已支持 `mock`、`qwen`、`deepseek`、`ollama`、`openai`，`/ai/chat` 和 `/audio/chat` 会返回 `assistant_text` 和 `llm` 信息。

- Firmware 已支持解析 `audio_url`、下载 WAV 并调用 `M5.Speaker.playWav()` 播放，PlatformIO 编译通过。
